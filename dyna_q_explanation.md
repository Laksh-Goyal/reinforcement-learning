# Dyna-Q with PyTorch

Welcome to the Dyna-Q implementation guide! 

Dyna-Q is an architecture that integrates **Learning** (improving the policy from real experience) and **Planning** (improving the policy from simulated experience generated by a learned model). 

When implementing Dyna-Q with deep learning (PyTorch), we are effectively building two neural networks (or families of networks):
1. **The Q-Network:** Predicts the action-values $Q(s, a)$.
2. **The Environment Model (Dynamics Model):** Predicts the next state $s'$ and reward $r$ given the current state $s$ and action $a$.

## The Core Loop of Dyna-Q

At each step in the environment:
1. **Direct Reinforcement Learning:**
   - Observe state $S$, choose action $A$ (e.g., using $\epsilon$-greedy based on your Q-Network).
   - Execute $A$ in the real environment, observe $S'$ and $R$.
   - Store the transition $(S, A, R, S', \text{done})$ in a Replay Buffer.
   - Train the **Q-Network** using standard DQN updates with a batch from the Replay Buffer.

2. **Model Learning:**
   - Train the **Environment Model** using the collected real experience to minimize the prediction error between its output $(\hat{S'}, \hat{R})$ and the actual $(S', R)$.

3. **Planning (The "Dyna" part):**
   - Perform $N$ planning steps:
     - Sample states and actions that have been previously experienced (or just states from the replay buffer and random actions).
     - Use the **Environment Model** to simulate the next state and reward: $\hat{S'}, \hat{R} = \text{Model}(S, A)$.
     - Train the **Q-Network** on these simulated transitions $(S, A, \hat{R}, \hat{S'})$ using the same DQN update rule.

## Why use Dyna-Q?
- **Sample Efficiency:** It squeezes more information out of every real interaction by simulating interactions in the "imagined" environment model.
- **Continuous Learning:** The model can be queried anytime, allowing the agent to plan without physically executing actions.

## Implementing with PyTorch

I have prepared a skeleton file for you at `algorithms/dyna_q/dyna_q.py`. 

### What you need to implement:
1. **The Q-Network (`QNetwork`)**: A standard MLP that takes a state and outputs Q-values for all actions.
2. **The Environment Model (`EnvironmentModel`)**: An MLP that takes a state-action pair and predicts the next state and the reward.
3. **The `optimize_q_network` function**: A standard DQN update using either real transitions from the Replay Buffer or simulated transitions.
4. **The `optimize_model` function**: A supervised learning update to train the Environment Model to predict $S'$ and $R$.
5. **The `planning_step` function**: The core of Dyna-Q. Sample states and actions, predict next states and rewards using the Environment Model, and then optimize the Q-network using these simulated transitions.

The `main()` loop is partially set up to guide you through the process of:
- Interacting with the environment.
- Training the Q-network with real experience.
- Training the Environment Model.
- Planning $N$ steps using the Environment Model to train the Q-network further.

Feel free to start filling in the skeleton! If you have any questions or get stuck on a specific part (like the loss functions or tensor shapes), just ask!
